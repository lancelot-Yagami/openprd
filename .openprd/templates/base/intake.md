# Intake 模板

## 发现

- 我们要解决什么问题？
- 这更像什么产品或能力？主要是给谁用的？如果你愿意，也可以直接说它更偏 consumer、b2b 还是 agent。
- 第一版最小可用切片，最希望先让用户完成什么？
- 当前临时解决方式是什么？
- 哪些内容这轮先不做？
- 哪些既有能力、数据、流程或体验不能被破坏？
- 成功指标是什么？
- 是否涉及登录、用户数据、多人协作、AI 调用、第三方服务、计费或其他会产生成本与风险的行为？

## Freeze Check

- 产品类型是否已明确？
- 关键角色是否已明确？
- 第一版切片是否已明确？
- 非目标和保护项是否已明确？
- 是否已有成功指标？
- 若存在消耗型成本，是否已明确额度限制、滥用防护、监控报警和止损动作？
- 是否存在阻塞 freeze 的开放问题？
