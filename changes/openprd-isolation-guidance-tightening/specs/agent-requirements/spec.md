## 新增需求

### 需求：OpenPrd 单独环境建议收敛
当前默认建议把不少续做场景放到单独环境里继续，这会额外增加分支、上下文切换和合并复杂度。

#### 场景：主流程成功
- **当** 用户继续已有任务时，系统回到正确上下文继续推进
- **则** 历史续做和指定任务续做默认回到对应上下文继续推进

#### 场景：边界情况保持可见
- **当** 跨工作区恢复时先切回正确工作区，但不默认追加单独环境建议
- **则** 产品应保持该情况明确可见，以支持实现和验证

#### 场景：失败模式得到处理
- **当** 推荐条件判断过宽会继续放大复杂度
- **则** 产品应提供有边界且可评审的结果
