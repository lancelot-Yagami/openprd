# OpenPrd 单独环境建议收敛

## 背景与原因

当前默认建议把不少续做场景放到单独环境里继续，这会额外增加分支、上下文切换和合并复杂度。

## 变更内容

- run context 推荐条件
- hook 与 CLI 的提示文案
- 相关回归测试
- 收紧单独环境建议的默认触发条件
- 识别用户是否明确要求隔离处理
- 同步收口 hook 和 CLI 对外提示
- 历史续做和指定任务续做默认回到对应上下文继续推进
- 只有长程 loop 或用户明确要求时才建议单独环境

## 能力范围

- `agent-requirements`: OpenPrd 单独环境建议收敛 需求。

## 影响范围

- 主要用户: 使用 OpenPrd 的 Agent
- 主要用户: 维护 OpenPrd 流程的操作者
- 成本来源: 主要成本来自额外分支、额外上下文和后续合并复杂度。
- 额度限制: 普通续做场景默认保持当前上下文处理，不主动扩展隔离环境。
- 依赖: 依赖当前 OpenPrd run、hook、review 与测试体系
- 风险: 历史续做与跨工作区恢复的推荐条件容易互相影响
- 风险: 提示文案和推荐结构如果不同步，用户感知会不一致
