## 新增需求

### 需求：工具自更新与项目升级编排
当前项目刷新命令只处理项目内生成物，用户还需要手动更新工具自身；本次要新增独立的工具自更新入口和组合升级入口，让工具更新后能够继续刷新单个旧项目或历史项目集群。

#### 场景：主流程成功
- **当** 用户运行 openprd self-update --dry-run，看到当前版本、目标来源和将执行的安装命令，不修改工具或项目。
- **则** 用户可以运行 openprd self-update 更新 CLI 自身。

#### 场景：边界情况保持可见
- **当** 当前运行环境是本地源码开发模式时，不自动覆盖源码，提示使用 npm install -g 或本地开发流程。
- **则** 产品应保持该情况明确可见，以支持实现和验证

#### 场景：失败模式得到处理
- **当** 安装命令失败时返回非零退出码并保留原项目不刷新。
- **则** 产品应提供有边界且可评审的结果
