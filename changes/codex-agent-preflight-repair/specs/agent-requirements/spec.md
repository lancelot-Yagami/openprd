## 新增需求

### 需求：Codex 代理启动前检查与显式修复
当本工具通过代码代理执行循环任务时，如果本机代理命令缺少平台原生组件，当前流程会直接暴露底层错误，任务无法进入子会话，用户也不知道如何修复。

#### 场景：主流程成功
- **当** 用户运行 Codex 代理循环；OpenPrd 先执行版本检查；成功后继续生成提示词并启动子会话。
- **则** 新增检查覆盖版本检查成功、组件缺失、修复成功和修复失败分支。

#### 场景：边界情况保持可见
- **当** Codex 命令不存在。
- **则** 产品应保持该情况明确可见，以支持实现和验证

#### 场景：失败模式得到处理
- **当** 修复失败时保留输出、错误、退出码和建议手动命令。
- **则** 产品应提供有边界且可评审的结果
